# v0.9 Draft — Data Foundation & Search

## Release name

Data Foundation & Search

## What's New

- Full-text search now targets concrete local text units instead of only aggregate cards.
- Conversation has an independent Note field.
- Local Asset Library metadata can be registered from Conversation Detail.
- A manual project backup script and Data Management guidance clarify retention boundaries.
- Task remains available but is visibly optional in the knowledge workflow.

## SearchDocument

`SearchIndexService` builds non-persistent SearchDocuments for Workspace, Conversation, Source, Message, derived Q&A Pair, Proposal, KnowledgeCard, Task, and Tag. Results include a snippet, source path, Workspace, matched fields, relevance score, and exact / contains / fuzzy mode. Filters cover entity type and Workspace.

## Conversation Notes

Conversation `note?: string` is backward compatible, editable with Save/Cancel, updates Conversation `updatedAt`, and is searchable without changing Source, Messages, Proposal, or Knowledge.

## Local Asset Library metadata

Asset Entity / Contract / BrowserStorage / Service store metadata only. Conversation Detail supports filename, local path, note, and metadata deletion. The browser does not upload, copy, or read files, and deleting metadata does not delete a local file.

## Backup script

`node scripts/backup-local-data.mjs` creates a timestamped directory under `backups/`; `--target` accepts a custom output directory. It copies docs, core project documents, and project-local `data/` when present. It does not follow external Asset paths or export browser LocalStorage.

## Data Management help

Settings and Help explain where browser records, repository files, and Asset metadata live, provide the backup command, and state the current export limitation.

## Known limitations

- Fuzzy matching is a simple normalized subsequence algorithm, not typo distance or semantic search. Short queries can produce broad low-score matches.
- Search indexes are rebuilt synchronously from LocalStorage and suit the current small single-browser dataset.
- Message/Q&A result navigation opens the owning Conversation but does not scroll to an anchored row.
- Asset paths are user-entered references and are not validated for existence.
- The backup script does not export browser LocalStorage or external files.
- No automated test suite or completed manual v0.9 QA is claimed.

## Non-goals

- RAG, embeddings, semantic or hybrid search
- Agent, Calendar, Reminder, recurring automation, or Task escalation
- Cloud Sync, object-storage upload, or real cloud Provider API
- Database or SQLite migration
- Browser filesystem scanning or automatic file copying

## Next release suggestions

- Design an explicit LocalStorage JSON export/import format with versioning, validation, preview, and rollback.
- Measure search quality and dataset size before choosing Fuse.js, MiniSearch, or a desktop SQLite FTS adapter.
- Add precise Message/Q&A anchors and optional Asset path integrity checks in a CLI/desktop adapter.
