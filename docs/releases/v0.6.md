## Architecture

系统保持 local-first 模块化单体：Entity 定义领域数据，Contract 描述能力，BrowserStorage 集中处理 LocalStorage，Service 编排规则与用例，Page 负责路由与交互。

核心链路保持不变：

```text
Import / Conversation
  → Source / Messages
  → AnalyzerProvider
  → Proposal
  → Human Review
  → KnowledgeCard / Tag
```

Demo Provider 默认可用且不访问网络。Ollama 默认关闭，只在用户配置、启用并选择后访问本地 HTTP 服务。任何 Analyzer 都只能生成 Proposal，不能直接创建 KnowledgeCard 或绕过 Review。

## Known Limitations

- 数据仅保存在当前浏览器 LocalStorage；没有正式备份、云同步、数据库或多用户能力。
- Search 使用同步线性扫描，适合小数据量，不提供索引、相关性评分、Embedding 或 RAG。
- 云 Provider 仅展示配置边界，不会发起真实调用，也不收集 API Key。
- Ollama 只支持可选的本地非流式调用；项目不负责安装服务或下载模型。
- Workspace 为单层归属，不支持目录树、权限或团队协作。
