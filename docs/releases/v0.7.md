# Release v0.7 — Daily Learning Workflow

- Release date：2026-07-04
- Phase：Phase2
- Status：Completed

## What's New

v0.7 adds a local-first daily action layer beside the existing knowledge workflow. Users can capture Tasks, organize them in Today and Tasks views, create traceable Tasks from source content, and find Tasks through Search 2.0.

## Task Domain

- Task is an independent entity with status, priority, type, due date, Workspace placement, optional SourceRef snapshots, and lifecycle timestamps.
- Task lifecycle actions include create, complete, reopen, archive, restore, and delete.
- Workspace deletion moves Tasks to Inbox; deleting a Conversation or Knowledge source does not delete linked Tasks.
- Browser persistence continues through TaskStorage and BrowserTaskStorage; Page and Service code do not access LocalStorage directly.

## Today UI

- `/today` groups Overdue, Today, Upcoming, Inbox, and Completed Today.
- Quick Capture supports Workspace, Task type, priority, and due date.
- `/tasks` provides complete lifecycle views and title/description filtering.

## Source-linked Task

- Knowledge Detail, Conversation Detail, and selected Messages can explicitly create a Task.
- SourceRef preserves identity plus title and summary snapshots.
- Missing sources degrade to `Source deleted`; the Task and its snapshots remain available.

## Task Search

- Search 2.0 now indexes Task title, description, SourceRef title snapshot, and SourceRef summary snapshot.
- `/search` provides a Task group, Task entity filter, Workspace filter, and Task status, priority, and type filters.
- Empty search can include Tasks in the 12 most recently updated local records.
- A Task result opens `/tasks?q=<task title>`. This restores a title query but is not ID-based exact focus, so same-title Tasks may appear together.
- Search URLs persist `q`, `type`, and `workspaceId`. Task status, priority, and Task type filters are session UI state only.

## Known Limitations

- Data remains in the current browser LocalStorage with no backup, sync, database, or multi-user support.
- Search is a synchronous linear scan without relevance scoring, index, embedding, or RAG.
- Task result navigation filters by title rather than scrolling to or highlighting one Task ID.
- Task-specific filters are not persisted in the URL.
- Workspace remains single-level.

## Non-goals

- Activity
- Calendar
- Reminder or notification
- Recurring Task
- Agent or autonomous Task mutation
- RAG, embedding, or database
- AI Suggest Task

## Next Release Suggestions

Evaluate Epic E — Knowledge Productivity with a separately approved scope and acceptance criteria. Candidate work should deepen review and reuse of knowledge without assuming Activity, Agent, RAG, Calendar, or automation is approved.
