# v1.0 Alpha — Second Brain Workspace

- Date: 2026-07-06
- Scope: Phase1 A–L + Phase2 M–AB
- Status: implementation complete; automated gates passed; manual QA pending
- Breaking changes: None intended

## Product model

- Conversation = 一个可长期追加的长对话线程。
- Round = 一轮问答，也是最小整理单位。
- Proposal = AI 整理建议 / 草稿；不会自动成为知识。
- Knowledge = 用户确认后的知识。
- Workspace / Folder = Conversation 的多层组织树。
- Recipe = 本地手动工作流模板，不是 Agent。
- Task / Today = 可选行动项，不是主流程。

## Highlights

- 多层 Workspace/Folder、Conversation Explorer、最近 20 条快速切换和三栏 Workspace Mode。
- Round Note/Summary、Conversation Summary/Conclusion/Pending Questions，以及 Round 多 Knowledge。
- Knowledge 更新建议需人工确认，并保留 previous content snapshot。
- 多轮粘贴解析、Manual Round Builder、Analyzer 状态/错误恢复和临时 Provider Selector。
- Search 结果提供命中字段、片段、层级路径和来源 Round anchor；fuzzy 不是语义搜索。
- Conversation/Round Asset metadata 支持 status、Relink 与 Mark Missing，不读取或删除真实文件。
- Recipe、Feedback、AppEventLog、Data Health、Cmd+K 与 Context Help。
- PALOS App Data JSON 导出/分类导入/失败回滚，以及实体 Markdown/JSON 导出。

## ChatGPT official export import

当前只支持 ChatGPT 官方 Export zip 解压后的 `conversations.json` 最小文本导入：

- 列出 Conversation title、支持的 Message 数、create/update time 和 unsupported 数。
- 沿 current-node 主分支线性化 User/Assistant 文本。
- 首次导入经现有 Parser/ImportService 创建 Conversation、Messages 与 Rounds。
- 重复导入复用 external Conversation ID；Message 优先按 external message ID，缺失时按 content hash 去重，只追加新 Message。
- 增量导入不自动覆盖旧 Rounds；用户需人工检查并按需 regenerate。
- 不支持完整 zip 自动解析、附件、图片、tool call、canvas、voice 或 shared link。

## Compatibility and safety

所有新增持久化字段均为 optional 或有读取归一化。旧 Conversation、Message、Round、Proposal、Knowledge、Workspace 与 Asset 不要求回填。删除 Folder 不删除 Conversation；App Data 导入先 Preview/二次确认，写入失败回滚。

没有数据库、RAG、Embedding、Agent、Calendar、Reminder、Cloud Sync、真实浏览器插件或真实 OpenAI/Claude API。Demo Provider 默认可用；Ollama 仅在用户显式配置后访问本地服务。
