# PALOS v1.1 Draft — Long Conversation UX & Import Stabilization

## Status

- Date: 2026-07-06
- Version: v1.1 alpha draft
- Epic C–J implementation complete；lint/build/diff-check checkpoints 已通过
- Git：未创建 commit
- 人工 QA：待执行

## What's New

### Conversation Navigator (Epic B 验收)
- 左侧可折叠 Round 导航面板
- 展开：显示全部 Round 序号、标题、Summary/Proposal/Knowledge 状态，支持搜索
- 折叠：仅显示 Round 序号圆点，hover 显示 tooltip
- 点击跳转到对应 Round

### Message Timeline 三态 (Epic C)
- **Collapsed**：默认模式，避免长对话直接展示全部消息
- **Preview**：显示前 5 条 Message，提供“展开全部”按钮与剩余数量提示
- **Full**：完整 Timeline，保留搜索、编辑、选择、Analyze 功能
- 三态通过 Collapsed / Preview / Full 按钮明确切换
- 与 Round Navigator 不冲突

### Round Inspector (Epic D)
- 在 Round 区域点击“Inspect”按钮打开右侧 Inspect 面板
- 显示并可编辑 Round Note 与 Round Summary
- 列出关联 Proposal / Knowledge / Assets
- Analyze 当前 Round 按钮
- “从当前 Round 创建 Knowledge”按钮
- 上一 Round / 下一 Round 快速切换
- 不实现复杂三栏最终版

### Import 2.0 (Epic E)
- 入口文案优化：
  - “粘贴并导入对话”（支持六种角色别名）
  - “📦 导入 ChatGPT Export”（独立突出，绿色边框）
  - “导入 TXT 文件”
- ChatGPT Export 入口视觉更突出
- Manual Round Builder 按钮改为醒目的“✋ 手动整理轮次”
- Role alias 文档明确列出：User/Assistant、用户/AI、我/GPT、问/答
- Shared link / 浏览器插件入口预留文本

### Search UX (Epic F)
- 搜索结果默认优先级：Conversation → Knowledge → Round → Proposal
- Raw Message 仅高级模式可见
- Round 结果新增“跳转到对应 Round →”深链
- 保留现有 fuzzy 行为，不引入新搜索库或语义搜索

### Error / Feedback (Epic G)
- Feedback 实体增加可选 `page` 字段
- Feedback 页面支持快速记录，自动捕获页面 URL
- Conversation Detail 增加“记录反馈”链接
- Data Health 增加：
  - duplicate import risk（共享 externalConversationId 的重复 Conversation）
  - orphan round（所属 Conversation 已删除）
  - 更详细的 missing sourceRoundId 说明

### Docs (Epic H)
- Help 页面增加 v1.1 新功能区域与全部新概念
- CHANGELOG 增加 v1.1 条目
- HANDOFF 完整记录 Epic C–J 实现
- QA_CHECKLIST 增加 v1.1 手动验收条目
- 新增本 Release Draft

## Known Limitations

- Message Timeline 三态仅在 Conversation Detail 页面生效；不跨页面
- Round Inspector 仅在 Classic Mode 的 Round Workspace 提供；Workspace Mode 已有类似功能
- Import 2.0 不改变现有 import pipeline 行为；不实现 shared link 抓取或浏览器插件
- Search 不实现语义搜索、Embedding 或 RAG
- Feedback 不上传任何数据；所有记录仅保存在浏览器 LocalStorage
- Data Health 为只读报告，不会自动修复问题
- 不做数据库迁移，不删除旧数据

## Architecture Impact

- Entity：Feedback 增加可选 `page` 字段，向后兼容
- Storage：BrowserFeedbackStorage.save() 接受可选 `page`
- UI：ConversationDetail、RoundWorkspace、ImportWorkbench、SearchExperience、FeedbackList、DataHealthReport、Help 修改
- Core domain（Round/Conversation/Knowledge/Proposal）无变更
- ChatGPT Import tests/fixtures 未修改
