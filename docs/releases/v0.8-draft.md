# Release v0.8 Draft — Product Onboarding + Conversation Q&A Pair UX

- Draft date：2026-07-04
- Phase：Phase2
- Status：Implementation complete / Manual QA pending

## Goals

v0.8 让已有本地知识工作流更容易理解和操作，重点覆盖 Product Onboarding、Provider Selection 和 Conversation Q&A Pair UX。

## Included

- `/help` 内置中文操作手册、核心概念与 Workspace → Import → Conversation → Generate Messages → Q&A Pair → Analyze → Review → Knowledge 推荐流程。
- Settings 只展示 Demo / Ollama；Ollama 必须 enabled 且最近 Connection Test Success 后才能设为当前 Provider。默认模型为 `qwen3:8b`。
- Q&A Pair 从 Messages 派生：User / Unknown 开组，连续 Assistant 合并，保留未回答与 Orphan Assistant Pair。
- Conversation 提供 Timeline / Q&A Pair 双视图、Pair 搜索、三种排序、展开折叠和按 Pair 多选 Analyze。
- 六步动态流程条根据原文、Messages、选择、Proposal 和 Knowledge 提示下一步。
- Knowledge 明确 Active / Archived 语义，推荐 Archive；Delete Forever 位于双重确认 Danger Zone。
- Dashboard 横向展示最近 8 条 Conversation 和 60 字内容摘要。

## Architecture and Compatibility

- Q&A Pair 是 Core 运行时派生模型和纯服务，不新增 BrowserStorage、LocalStorage key 或迁移。
- Pair Analyze 映射为现有 selected Message IDs，继续复用 Analyzer → Proposal → Review → Knowledge 流程。
- Provider 当前选择仍通过 AIProviderStorage 保存；Provider Service 同时校验 Ollama enabled 与 Test Success。
- Sprint1–Sprint6 旧数据缺失 `updatedAt` 时继续由 MessageStorage 回退 `createdAt`。

## Non-goals

- RAG、Elasticsearch、Embedding
- Agent、Calendar、Reminder、催办系统
- 数据库或云 Provider API
- Q&A Pair 手动拆分、合并、拖拽排序或独立持久化
